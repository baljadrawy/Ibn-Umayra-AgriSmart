# التوثيق التقني الشامل - نظام التقويم الزراعي المطور (ابن عميرة)

هذا المستند يشرح بنية النظام، المنطق البرمجي، وكيفية عمل المكونات المختلفة في مشروع "ابن عميرة AgriSmart".

---

## 1. المعمارية التقنية (Technical Architecture)
النظام مبني باستخدام تقنيات حديثة تضمن السرعة والدقة:
- **إطار العمل (Framework):** Next.js 15 (App Router) مع React 19.
- **التنسيق (Styling):** Tailwind CSS مع مكونات Shadcn UI لضمان واجهة مستخدم متوافقة مع معايير Apple (Clean & Modern).
- **قاعدة البيانات (Database):** Firebase Firestore لتخزين سجلات المزرعة وبيانات الأنواء.
- **الذكاء الاصطناعي (AI):** Firebase Genkit مع نموذج Google Gemini 2.5 Flash لمعالجة الاستشارات الزراعية.
- **الطقس (Weather API):** Open-Meteo API لجلب البيانات اللحظية بناءً على الإحداثيات.

---

## 2. قاعدة البيانات المركزية (Core Data Engine)
يقع "عقل" النظام في ملف `src/lib/location-data.ts` الذي يحتوي على:
- **تقويم 2026:** هيكل بيانات يضم 30 نوءاً (نجماً) مع مواعيد دخولها وخروجها بدقة ميلادية وهجرية.
- **الأحزمة المناخية الخمسة:** تعريف المناطق الجغرافية ومعاملات التعديل (Offsets) الخاصة بها.
- **قاعدة بيانات المدن:** أكثر من 60 مدينة ومحافظة سعودية مع إحداثياتها الجغرافية.

---

## 3. منطق التعديل المكاني الذكي (Location Adjustment Logic)
بما أن تقويم ابن عميرة مبني أصلاً على مناخ "الطائف"، يقوم النظام بتعديل المواعيد آلياً للمناطق الأخرى كالتالي:
- **المنطقة الوسطى (الرياض، القصيم):** إضافة (+7 أيام) للمواعيد الشتوية بسبب اختلاف برودة التربة.
- **المنطقة الشمالية (تبوك، حائل):** إضافة (+14 يوماً) بسبب طول موسم الصقيع.
- **المنطقة الشرقية (الدمام، الأحساء):** تقديم المواعيد (-10 أيام) بسبب الدفء والرطوبة المبكرة.
- **المنطقة الغربية (جدة، جازان):** تقديم المواعيد (-7 أيام) مع دعم محاصيل استوائية (مانجو، بابايا).

---

## 4. المكونات الرئيسية للنظام (System Components)

### 4.1 نظام التهيئة (Smart Onboarding)
- **الملف:** `src/app/page.tsx` (Dialog Component).
- **الوظيفة:** يظهر للمستخدم الجديد ويطلب تحديد الموقع (آلياً عبر GPS أو يدوياً).
- **الحفظ:** يتم تخزين الاختيار في `localStorage` لضمان تجربة مخصصة عند العودة.

### 4.2 محرك مقارنة الطقس (Weather Comparison Engine)
- **المكون:** `src/components/dashboard/WeatherCompare.tsx`.
- **الوظيفة:** يجلب حرارة المدينة المختارة، يقارنها بـ "الحرارة المتوقعة" في تقويم ابن عميرة، ويحسب "نسبة التوافق" (Match Score).

### 4.3 نظام التوصيات الديناميكي (Dynamic Recommendations)
- **المكون:** `src/components/dashboard/RecommendationList.tsx`.
- **المنطق:** يدمج بيانات النوء الحالي مع الحزام المناخي والحرارة الحقيقية. إذا كانت الحرارة الحقيقية أعلى من المتوقع بـ 5 درجات، يضيف النظام نصيحة "كثف الري المسائي".

### 4.4 المستشار الزراعي الذكي (AI Advisor)
- **المسار:** `src/ai/flows/ask-ai-agent-agricultural-advice-flow.ts`.
- **الوظيفة:** وكيل ذكي "يعرف" مكان المستخدم والنوء الحالي. عند سؤاله "متى أزرع الطماطم في جدة؟"، يقوم المحرك بحساب التوقيت بناءً على حزام جدة ويجيب باللغة العربية.

---

## 5. معايير البيانات (Data Standards)
- **الصور:** يتم جلب صور المحاصيل من ملف `placeholder-images.json` مع كلمات مفتاحية تضمن ظهور صور "صحية" ونضرة.
- **الأمان:** استخدام `firestore.rules` لضمان خصوصية بيانات المزارعين (في حال تفعيل الحسابات).

---

## 6. المصادر المعتمدة
1. تقويم ابن عميرة الزراعي المناخي 2026.
2. بوابة البيانات المفتوحة لوزارة البيئة والمياه والزراعة (MEWA).
3. المركز الوطني للأرصاد (NCM).
4. Open-Meteo API.
